O que é este sistema

Uma demonstração de Intrusion Detection por fluxo. O backend executa um modelo de Machine Learning treinado com features do CICFlowMeter sobre os datasets UNSW-NB15 e CIC-IDS2017 para estimar se os fluxos enviados são benignos ou maliciosos e sumariza a distribuição. É voltado para estudo, prototipagem e homelab.

Limitação: a inferência foi validada com dados de fluxo compatíveis com CICFlowMeter (CIC-IDS2017) e mapeamento de colunas do UNSW-NB15. Outros formatos podem exigir adequação de features e re-treino para resultados confiáveis.

Como funciona

  1. Você envia um .pcap/.pcapng ou .csv via POST /predict.
  2. Se for PCAP, o servidor usa o CICFlowMeter para extrair features de fluxo.
  3. O script predict.py carrega o modelo intrusion_model.joblib e aplica o mesmo pré-processamento do treino.
  4. Cada fluxo é classificado como BENIGN ou MALICIOUS.
  5. As contagens são agregadas e exibidas: total de fluxos, BENIGN, MALICIOUS e % malicioso.
  6. Um gráfico donut resume a distribuição e a seção de insights sugere próximos passos.

Compatibilidade: use CSVs gerados pelo CICFlowMeter (CIC-IDS2017) ou o esquema do UNSW-NB15. PCAPs enviados são convertidos pelo CICFlowMeter para o mesmo conjunto de features antes da predição. Outros datasets podem falhar por incompatibilidade de colunas.

Privacidade: o arquivo enviado é apagado do servidor após o processamento. Resultados dependem do domínio, das colunas disponíveis e da qualidade da captura.
